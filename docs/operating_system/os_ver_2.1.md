# HyperProof Lab Operating System v2.1 (Audio-First Pivot)

**“AI가 문제를 푼다. 인간은 문제를 정의한다.”**

**v2.1 | 2026.01.14 | HyperProof Lab**

---

## 0) About

HyperProof Lab은 AI의 한계를 매일 밀어붙이는 사람들입니다. 개발, 전략, 리서치—각자의 도메인에서 AI가 실제로 무엇을 바꾸는지 경험하고, 그 관찰을 기록합니다.

**우리가 하는 일**

1. **리서치 & 아카이브**: Frontier AI의 변화를 추적하고 “무엇을 의미하는가”를 관찰/기록합니다.
2. **콘텐츠(브랜딩)**: 일반인이 이해할 수 있는 언어로 번역해, “AI를 잘 쓰는 팀”을 증명합니다.
3. **교육(검증→캐시)**: 아이들과 일반인이 “문제정의→가설→검증”을 하도록 돕습니다. 코딩은 수단입니다.

---

## 0.5) 운영 전제 (Production Constraints)

**원칙: 지금 가능한 제작 조건에서 시작한다.**

- 장비/스튜디오/인력이 갖춰졌다는 가정으로 시스템을 설계하지 않습니다.
- “최소 제작비 + 최소 준비시간 + 빠른 발행”을 최우선 최적화합니다.
- 초기 품질 목표는 “완벽”이 아니라 **학습 가능한 최소 완성본(MVP)** 입니다.

## 1) 비즈니스 모델 (Audio-First로 수정)

```
[브랜딩] 오디오 팟캐스트 → “AI 잘 쓰는 팀” 반응으로 증명
            ↓
[검증] 산스 키즈스쿨 파일럿 (10명, 저가/무료)
            ↓
[캐시] 대치동 바이브코딩 (인당 100만원/주말)
            ↓
[스케일] 투자 유치 → 영상/비주얼 확장 + 글로벌 컴피티션

```

**투자 논리(핵심 메시지)**

- “음성만으로도 이 정도 반응”을 먼저 증명하고,
- 투자 이후 **영상/비주얼 확장**으로 콘텐츠 폭발력을 키웁니다.

**CapEx 원칙**

- 장비/스튜디오 등 확장 비용은 **개인 비용으로 처리하지 않고**, 투자 이후 **법인 비용**으로 집행합니다.

## 3) 트랙 A: 콘텐츠 (브랜딩) — Audio-First MVP

### 3.1 목적

- “AI를 잘 쓰는 팀”을 **빠르게** 증명
- 일반인 대상, 대중 언어로 번역된 인사이트 생산

### 3.2 운영 방식 (핵심 변경)

**주제만 정하고, 의식의 흐름대로 대담을 진행합니다.**

- 무엇이 튀어나올지 모르는 “불확실성”을 콘텐츠 엔진으로 사용합니다.
- 스토리보드 중심이 아니라 **주제 중심**으로 운영합니다.

### 3.3 준비물 최소화 (Minimal Prep Rule)

에피소드마다 사전 준비는 아래 3가지만 필수입니다.

1. **주제 1개**
2. **시작 질문 3개**
3. **Red Lines 리마인드**

그 외(자료/시연/레퍼런스)는 “있으면 좋음”이며, **발행을 막는 조건이 아닙니다.**

## 3.4 시즌 1: “당신은 대체될 것이다” (유지, 톤은 라이트)

**컨셉**

- AI가 이 직업을 대체할 수 있나? 공격과 방어
- 시즌당 5회 (초기에는 “가볍게 완주”가 1순위)

**포맷**

- 진행 1 + 패널 2 (고정)
- 게스트는 Phase 2부터 본격 확장(초기에는 내부 포맷 안정화 우선)

| 시간 | 섹션 | 내용 |
| --- | --- | --- |

## 3.6 오디오 → 숏폼 재가공 파이프라인 (Repurpose Rule)

- 오디오 원본에서 **하이라이트 3개(각 30~60초)** 를 추출합니다.
- 쇼츠/릴스는 “영상 촬영”이 아니라 **오디오 기반 자막형**을 기본으로 합니다.
- 초기 목표는 “정교한 편집”이 아니라 **빈도와 학습**입니다.

## 5) 내부 리서치 시스템 (부담 최소 유지)

### HypeProof Lens (확신도 라벨링)

- **Observed**: 직접 사용/관찰로 확인
- **Supported**: 1차 소스로 뒷받침
- **Speculative**: 해석/추정
- **Unknown**: 정보 부족

### Claim 카드 (주간 2~3개)

```markdown
- Claim: 판정 가능한 문장 1개
- Label: Observed / Supported / Speculative / Unknown
- Evidence: 링크/관찰/근거
- Boundary: 어디서 깨지나
- Next Question: 다음 주 확인할 질문

```

### 운영 원칙

1. **부담 최소화**: 매주 2~3개면 충분
2. **재사용 가능 형태**: 나중에 찾을 수 있게
3. **콘텐츠와 느슨한 연결**: 필요할 때만 사용
4. **콘텐츠 문장 규칙**: 외부 발언은 최소 Supported 이상을 권장(Observed 우선)

## 10) 브랜드 언어 (유지)

- **HyperProof** = “AI가 문제를 푼다. 인간이 정의한다”를 증명
