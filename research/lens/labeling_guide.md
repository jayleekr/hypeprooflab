# HypeProof Lens - 확신도 라벨링 가이드

> AI 관련 주장에 대한 확신도를 체계적으로 분류하는 시스템

---

## 라벨 정의

### 🟢 Observed (관찰됨)

**정의**: 직접 사용/관찰로 확인한 사실

**기준**:
- 팀원이 직접 테스트/사용해봄
- 결과를 재현할 수 있음
- 1차 경험에 기반

**예시**:
- "Claude가 10,000토큰 이상의 코드를 한 번에 생성할 수 있다" (직접 테스트 후)
- "GPT-4가 특정 수학 문제에서 오류를 냈다" (직접 확인)

---

### 🔵 Supported (뒷받침됨)

**정의**: 1차 소스로 뒷받침되는 주장

**기준**:
- 공식 문서, 논문, 발표에서 확인
- 신뢰할 수 있는 출처 존재
- 직접 확인하진 않았지만 근거가 명확

**예시**:
- "Anthropic이 Claude 3.5를 발표했다" (공식 블로그)
- "GPT-4의 컨텍스트 윈도우는 128K이다" (OpenAI 문서)

---

### 🟡 Speculative (추정)

**정의**: 해석/추정에 기반한 주장

**기준**:
- 기존 데이터에서 추론
- 아직 검증되지 않음
- 논리적이지만 확정되지 않음

**예시**:
- "이 추세라면 2년 내 AGI가 가능할 것이다"
- "이 기능이 추가되면 X 산업이 변할 것이다"

---

### ⚪ Unknown (미확인)

**정의**: 정보 부족으로 판단 불가

**기준**:
- 근거 자체가 없음
- 상충하는 정보가 있음
- 추가 조사 필요

**예시**:
- "X 회사가 Y 기술을 개발 중이다" (루머)
- "이 벤치마크가 실제 성능을 반영하는지 불명확"

---

## 콘텐츠 문장 규칙

외부 발언(팟캐스트, 숏폼 등)에서는:

1. **Observed 우선**: 직접 확인한 것 위주로 발언
2. **최소 Supported**: 확인 안 된 건 "~라고 알려져 있다" 표현
3. **Speculative 명시**: 추정일 경우 "추정이지만", "아직 확실치 않지만" 표현
4. **Unknown 회피**: 정보 부족한 주제는 언급 자제

---

## 라벨 결정 플로우

```
직접 테스트/경험했나?
  ├─ Yes → 🟢 Observed
  └─ No → 공식 소스가 있나?
            ├─ Yes → 🔵 Supported
            └─ No → 논리적 추론인가?
                      ├─ Yes → 🟡 Speculative
                      └─ No → ⚪ Unknown
```
